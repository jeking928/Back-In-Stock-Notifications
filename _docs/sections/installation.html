Installation

Installation of the module has been made as simple as possible and shouldn't take long for any stores which have not already modified any of the template files which this module has to modify.

If any of the template files this distribution uses have already been modified (because of another third party module being installed), then the changes for the respective files should be applied manually, by copying across the appropriate sections of code required for the Ceon Back In Stock Notifications module. These sections are clearly marked (with “// BEGIN” and “// END” comments) within the sample files provided in this distribution.

Installation Overview

  1. Copy the new files from the distribution into the Zen Cart installation.
  2. Copy the updated core file from the distribution into the Zen Cart installation, or modify the respective core file.
  3. Copy the updated template files from the distribution into the Zen Cart installation, or modify their equivalents in the store's existing template files.
  4. Create the database table & add the admin configuration options, by running the provided SQL script.
  5. Copy the e-mail templates to a directory, for each language the store uses.
  6. [Optional but Recommended] Prevent the module's files from being indexed by search engines.

  2. Updated Core File

    A core Zen Cart file, functions_general.php, must be updated to display the Subscription Notification link for sold out products on the various product listing pages.

    • It is possible that this file has already been modified for the store, in which case the change within the sample file, included in this distribution for the store's Zen Cart version, must be applied to the previously modified core file.

      The change is clearly marked within the sample file in the distribution and should be easy enough to copy across, especially if a “diff” software package (such as the free WinMerge software or the excellent, commercial Beyond Compare software) is used.

      The change begins with a comment:

      // BEGIN CEON BACK IN STOCK NOTIFICATIONS 1 of 1

      And ends with a comment:

      // END CEON BACK IN STOCK NOTIFICATIONS 1 of 1

      • The change should be copied and pasted into functions_general.php at the exact same place it is present within the respective sample file.

    • If this file has not already been modified for the store, the pre-modified sample file from this distribution, for the version of Zen Cart being used by the store, can simply be copied over the store's existing version of the file.

      As with the new files, the store's admin directory may have been renamed from the default of “admin”, so be careful to copy the file into the correct folder!

    The modified core file can be found within the modified_core_files folder in the distribution:

    • includes/functions/functions_general.php

  3. Template Files For Zen Cart 1.3.x

    Modifications have been made to “clean” copies of the template files from a “fresh” installation of Zen Cart and are included as “sample” files in this distribution.

    • If any of these template files haven't already been modified for the store, the sample files included in this distribution can be copied into the store's template directory and will then override the standard versions of the files included in the Zen Cart distribution.

    • If any of these template files have already been modified for the store, then the changes necessary for the Back In Stock Notifications module to function must be made to the store's modified file(s).

      The changes are clearly marked within the sample files in the distribution and should be easy enough to copy across, especially if a “diff” software package (such as the free WinMerge or the excellent, commercial, Beyond Compare) is used.

  4. Create the Database Table & Add the Admin Configuration Options

    This software comes with some intelligent installation and error checking functionality, which makes it easy to install the module and to keep the database and configuration up to date.

    This installation/upgrade functionality runs automatically when the admin utility is accessed. (In the future, if the module is being upgraded, the admin utility should also be accessed then.)

    • Simply go to Catalog > Ceon Back In Stock Notifications in the Admin and the database table and configuration settings will be automatically created.

  5. Copy the E-mail Templates to a Folder for Each Language the Store Uses

    Zen Cart's e-mail functionality requires that there are e-mail template files for every language the store uses, in order to create HTML versions of any e-mails being sent.

    The five Back In Stock Notifications e-mail templates should therefore be copied into a folder for each language the store uses.

    For each language the store uses carry out the following steps:

    • If one doesn't already exist, create a folder for the language within the e-mail templates folder.

      The name of the folder is simply the two letter language code for the language.

      For example, if the store uses Spanish, the language code for which is “es”, the following folder should be created in the store's e-mail folder:

      email/es

    • Copy the five Back In Stock Notifications e-mail template files from the email templates folder to the language's e-mail templates folder.

      For example, copy email/email_template_back_in_stock_notification.html to email/es/email_template_back_in_stock_notification.html

    • The e-mail templates can now be customised for the language if desired, simply by editing the files in the language's e-mail templates folder.

  6. [OPTIONAL but RECOMMENDED] Prevent the module's files from being indexed by search engines.

    It's unlikely that anyone would want a search engine (web robot) indexing the subscription/unsubscription pages, as it could allow people to subscribe to a Back In Stock Notification List for a product that is actually back in stock.

    This is easily avoided by using Zen Cart's “ROBOTS_PAGES_TO_SKIP” define:

    • The following two “page names” should be added to the define for ROBOTS_PAGES_TO_SKIP in the file includes/languages/LANGUAGE/meta_tags.php (for each language the site uses):
    • back_in_stock_notification_subscribe,back_in_stock_notification_unsubscribe
    • Please Note: A comma should be added before “back_in_stock_notification_subscribe” within the define!
