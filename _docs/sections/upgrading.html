Downloading the Latest Version

The latest version of the module distribution can always be found on its website, http://dev.ceon.net/web/zen-cart/back-in-stock-notifications/downloads.

Upgrading

Simply follow the instructions for the version of Back In Stock Notifications already installed..

  1. Upgrading to Version 3.2.0 from Version 3.0.x
  2. Upgrading to Version 3.2.0 from Version 2.6.0 or Version 2.4.0
  3. Upgrading to Version 3.2.0 from Version 2.0.0 - 2.2.4

Upgrading to Version 3.2.0 from Version 3.0.x

There are quite a few files which have changed between these versions, but upgrading is straightforward:

  1. New/Updated Files

    Most files are used by all Zen Cart versions.

    There is an additional file which is used only by Zen Cart 1.3.x; if the store uses Zen Cart 1.3.x, be sure to copy this extra file across from the folder files/additional_file_for_1.3.x.

    Updated files for the Admin directory

    Most of the admin files have been updated.

    Overwrite the existing admin files with the new/updated files from version 3.2.0.

    Most Zen Cart stores have their admin directory renamed from the default of “admin”.

    Copy all of the folders and files from the files/admin folder into the store's “admin” directory, whatever it is called.

    The files are already in the correct directory structure, so a single drag-and-drop of the folders/files in the files/admin folder into the “admin” directory for the store will put the files in the correct locations.

    • admin/images/ceon-tab-background-selected.png
    • admin/includes/classes/class.CeonBISNInstallOrUpgrade.php
    • admin/includes/extra_datafiles/back_in_stock_notifications_database_tables.php
    • admin/includes/extra_datafiles/back_in_stock_notifications_filenames.php
    • admin/includes/functions/extra_functions/back_in_stock_notifications_admin_page_reg.php
    • admin/includes/functions/back_in_stock_notifications_functions.php
    • admin/includes/languages/english/extra_definitions/back_in_stock_notifications.php
    • admin/includes/languages/english/back_in_stock_notifications.php
    • admin/back_in_stock_notifications.php
    Updated File for Zen Cart 1.3.x

    Copy the file from the files/additional_file_for_1.3.x/admin folder into the store's “admin” directory, whatever it is called.

    • admin/includes/boxes/extra_boxes/back_in_stock_notifications_catalog_dhtml.php
    Updated files for the main store directory

    Most of the files for the main store directory have been updated.

    Copy the includes folder from the files folder into the store's main directory.

    The files are already in the correct directory structure, so a single drag-and-drop of the folder into the root directory for the store will put the files in the correct locations.

    Please do not copy the files individually - there's simply no need - doing so often results in files being missed out!

    Warning: Be careful NOT to copy the files from the files/includes folder into the store's admin/includes directory, they go in the includes directory which is alongside, not inside the store's admin directory.

    • includes/auto_loaders/config.back_in_stock_notificationsAccount.php
    • includes/auto_loaders/config.back_in_stock_notificationsProductInfo.php
    • includes/classes/observers/class.back_in_stock_notificationsAccount.php
    • includes/classes/observers/class.back_in_stock_notificationsProductInfo.php
    • includes/classes/class.CeonEmailValidation.php
    • includes/classes/class.CeonXHTMLHiTemplate.php
    • includes/extra_datafiles/back_in_stock_notifications_database_tables.php
    • includes/extra_datafiles/back_in_stock_notifications_filenames.php
    • includes/languages/english/extra_definitions/back_in_stock_notifications.php
    • includes/languages/english/account_back_in_stock_notifications.php
    • includes/languages/english/back_in_stock_notification_subscribe.php
    • includes/languages/english/back_in_stock_notification_unsubscribe.php
    • includes/modules/pages/account_back_in_stock_notifications/header_php.php
    • includes/modules/pages/back_in_stock_notification_subscribe/header_php.php
    • includes/modules/pages/back_in_stock_notification_unsubscribe/header_php.php
    • includes/templates/template_default/templates/tpl_account_back_in_stock_notifications_default.php
    • includes/templates/template_default/templates/tpl_back_in_stock_notification_subscribe_default.php
    • includes/templates/template_default/templates/tpl_back_in_stock_notification_unsubscribe_default.php

  2. Updated Core File

    A core Zen Cart file, functions_general.php, must be updated.

    • It is likely that this file has already been modified for the store, in which case the change within the sample file, included in this distribution for the store's Zen Cart version, must be applied to the previously modified core file.

      The change is clearly marked within the sample file in the distribution and should be easy enough to copy across, especially if a “diff” software package (such as the free WinMerge software or the excellent, commercial Beyond Compare software) is used.

      The change begins with a comment:

      // BEGIN CEON BACK IN STOCK NOTIFICATIONS 1 of 1

      And ends with a comment:

      // END CEON BACK IN STOCK NOTIFICATIONS 1 of 1

      • The change should be copied and pasted into functions_general.php at the exact same place it is present within the respective sample file.

    • If this file has not already been modified for the store, or the only time it was was for an earlier version of Ceon Back In Stock Notifications, the pre-modified sample file from this distribution, for the version of Zen Cart being used by the store, can simply be copied over the store's existing version of the file.

      As with the new files, the store's admin directory may have been renamed from the default of “admin”, so be careful to copy the file into the correct folder!

    The modified core file can be found within the modified_core_files folder in the distribution:

    • includes/functions/functions_general.php

  3. Updated BISN Template Files

    If the any of the following files exist in the store's templates directory, they must be updated, as the files have been updated for version 3.2.0.

    • includes/templates/STORE-TEMPLATE/templates/tpl_account_back_in_stock_notifications_default.php
    • includes/templates/STORE-TEMPLATE/templates/tpl_back_in_stock_notification_subscribe_default.php
    • includes/templates/STORE-TEMPLATE/templates/tpl_back_in_stock_notification_unsubscribe_default.php

    The files in the store's template templates folder must be updated to use the latest version of the code, present in the standard version of the files, which can be found in the files/includes/templates/template_default/templates folder in the distribution.

    • Copy the relevant updated code from the standard version of the file to the store's version of the file.

    • Using a “diff” software package, such as the free WinMerge or the excellent, commercial, Beyond Compare, makes this considerably easier to do without mistakes.

    • For example, compare the file includes/templates/STORE-TEMPLATE/templates/tpl_account_back_in_stock_notifications_default.php to includes/templates/template_default/templates/tpl_account_back_in_stock_notifications_default.php and change things appropriately.

  4. Updated Zen Cart Template Files

    Some of the standard Zen Cart template files which require modifications to work with this module have changed and need to be updated.

    Modifications have been made to “clean” copies of the template files from a “fresh” installation of Zen Cart and are included as “sample” files in this distribution.

    • If any of these template files haven't already been modified for the store, the sample files included in this distribution can be copied into the store template directory and will then override the standard versions of the files included in the Zen Cart distribution.

    • If any of these template files have already been modified for the store, then the changes necessary for the Back In Stock Notifications module to function must be made to the store's modified file(s).

      The changes are clearly marked within the sample files in the distribution and should be easy enough to copy across, especially if a “diff” software package (such as the free WinMerge or the excellent, commercial, Beyond Compare) is used.

      Instructions on how to copy across the modifications necessary for this module - to template files which have previously been modified - can be found in this FAQ.

    The sample template files can be found within the template_override_files folder in the distribution.

    The ones that have changed for version 3.2.0 - and therefore require updating - are:

    • includes/templates/STORE-TEMPLATE/templates/tpl_document_general_info_display.php
    • includes/templates/STORE-TEMPLATE/templates/tpl_document_product_info_display.php
    • includes/templates/STORE-TEMPLATE/templates/tpl_product_book_info_display.php
    • includes/templates/STORE-TEMPLATE/templates/tpl_product_free_shipping_info_display.php
    • includes/templates/STORE-TEMPLATE/templates/tpl_product_info_display.php
    • includes/templates/STORE-TEMPLATE/templates/tpl_product_music_info_display.php

      The modified e-mail template files can be found in the files/email folder in the distribution:

      • email/email_template_back_in_stock_notification_subscribe.html
      • email/email_template_back_in_stock_notification_subscribe.txt
      • email/email_template_back_in_stock_notification_subscribe_extra.html

    • However, if the store has already completely customised its e-mail templates, then this step may not be necessary, but the following information should be useful...

      Previously the greeting was hard-coded into the e-mail templates listed below. In this new version, a new variable is available to be used instead:

      $GREETING

      This can simply be placed the above variable at the point in the e-mail templates at which the greeting (e.g. “Dear”) should be placed, instead of hard-coding it in the e-mail template.

  6. Make sure there are e-mail templates for each language the store uses

    This wasn't specifically mentioned as an installation step in previous versions of the software, but it is completely essential.

    If the store uses more than one language, read and, if necessary, follow this section of the installation documentation.

      This can simply be placed the above variable at the point in the e-mail templates at which the greeting (e.g. “Dear”) should be placed, instead of hard-coding it in the e-mail template.

  6. Make sure there are e-mail templates for each language the store uses

    This wasn't specifically mentioned as an installation step in previous versions of the software, but it is completely essential.

    If the store uses more than one language, read and, if necessary, follow this section of the installation documentation.
