Configuration/Usage

No configuration is necessary with a default installation, but most users would agree that it's a good idea to style the e-mail templates etc. so that they match the store's styling.

Configuration

The module can be configured through the Zen Cart admin.

  • Go to Configuration > Ceon Back In Stock Notifications.
  • The only options are to enable/disable the functionality and to set an optional e-mail address to which copies of any Back In Stock Notifications should be sent.

Modifying the Plain Text E-mail Templates

The e-mail templates use basic HTML and can be found in the email folder in the store.

  • Aside from the default language, each language the store uses should have its own folder within this folder, as described in the installation/upgrading instructions.
  • There are HTML templates for each e-mail. ending in .html.
  • There are also additional e-mail template files ending in .txt, which are used as the basis of the plain text version of the e-mails and so can be customised separately!

Modifying the HTML Templates used by the BISN functionality

The HTML Templates, for the most part, don't use the standard Zen Cart templating methodology.

This module was originally part of an original version of Ceon's Site Engine system and makes use of our own, we believe (of course), far-superior templating system. Since it is released as a free gift to the Zen Cart community, it was deemed that it wasn't worthwhile downgrading the code to using Zen Cart's templating system.

  • To modify the look/layout of the code produced by this module the template file should be copied to the site's template override folder:

    includes/templates/template_default/templates/inc.html.back_in_stock_notifications.html

    Should be copied to:

    includes/templates/STORE-TEMPLATE/templates/inc.html.back_in_stock_notifications.html

  • This is a standard HTML file which has various “Template Parts” defined within.

  • Simply modify the HTML within this file and the changes will be instantly used by the module within the site.

  • Looking at the source, it can be seen that the template system uses tags to wrap blocks of source code which will be replaced with “live” values when the module is run and the template is used. It should be fairly obvious to anyone familiar with HTML how this can be used. Some hints follow:

    • Dynamic variables replace the contents of <ceon:variable name="variable_name"> </ceon:variable> tags. The names of the variables which will replace the contents of these tags should make their purpose obvious.
    • Entire sections of the source can be displayed or not, depending on whether or not a variable is set by the template. These sections are marked by <ceon:if isset="variable_name"> </ceon:if> tags.
    • All unused tags in the “ceon” namespace, and all unused “bracketed” tags (‘{ceon: }’) are stripped from the source before it is finalised - so there's nothing wrong with leaving lots of them in!

    We'd really like thoughts or feedback on the templating system: Please get in contact and let us know what you think!

Modifying the Format of the Product Listing Pages Subscription Links

The HTML for the links to the subscription form on Product Listing pages (category listings, featured product listings etc.) is specified in a define in the language file.

To change the link's format, a copy of the language definition file for the module must be made for each language the store uses.

As with all other language definition overrides in Zen Cart, this modified copy should be stored in the template directory for each language.

  • Copy the file includes/languages/english/extra_definitions/back_in_stock_notifications.php to the store's language templates folder. (E.g. includes/languages/english/extra_definitions/STORE-TEMPLATE/back_in_stock_notifications.php).
  • Make the necessary changes to this copy of the file.

Usage

The module comes with a handy Zen Cart admin utility.

  • Go to Catalog > Ceon Back In Stock Notifications.

As mentioned in the list of features, various options are available in this utility:

  • The current list of products with subscriptions can be viewed. This allows the store owner to quickly and easily see which products customers are interested in as well as the current stock level for each of these products.
  • The current list of products with subscriptions can also be viewed with additional information about the customers subscribed and when they subscribed.
  • A “Test Run” of the subscription notification mailing can be run. This will let the store owner see who would have been e-mailed, and about which product(s). The first e-mail that would have been sent is sent to the store owner so that they check if the e-mail is formatted correctly before the actual run!
    • Notifications are a one-time only action, so this helps avoids problems of sending illegible notification e-mails and to catch any errors in the e-mail (formatting mistakes, spelling mistakes etc).
  • The actual, real run of the subscription notification mailing can be run.
    • Once the notifications have been sent for real the subscriptions are permanently deleted!
